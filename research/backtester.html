Backtester¶

The backtester is the final step in the basic research workflow. It consumes a transaction dataframe and returns performance statistics, daily returns, trade-level records, an enriched bar dataframe, and warnings.

For strategies without stop logic, use haymaker.research.no_stop() to convert a dataframe containing position or blip into the transaction schema expected by haymaker.research.perf().

For notebook work, haymaker.research.auto_perf() is a convenience wrapper that accepts either an already prepared transaction dataframe or a raw strategy dataframe that can be passed through haymaker.research.no_stop().

Primary API¶

haymaker.research.no_stop(df, price_column='open')[source]¶

Convert researcher signals into a transaction DataFrame for perf().

Accepts a DataFrame with either a position column or a blip column (and optionally a close_blip column).

Dispatch precedence:

  • position column: used directly.

  • blip column: treated as generated events and converted to a next-bar position series.

  • Neither present: raises ValueError.

Parameters:

  • df (DataFrame) – Input DataFrame. Must contain price_column and either position or blip.

  • price_column (str) – Column used for execution prices and mark-to-market.

Returns:

bar_price, position, open_price, close_price, stop_price.

Return type:

DataFrame with columns

haymaker.research.perf(data, slippage=1.5, skip_last_open=False, raise_exceptions=True, use_numba=True)[source]¶

Return performance statistics for a strategy.

Parameters:

  • data (DataFrame) – Output of no_stop() or stop_loss() with bar_price added. Must have columns: bar_price, open_price, close_price, stop_price, position.

  • slippage (float) – Transaction cost expressed as a multiple of min tick.

  • skip_last_open (bool) – If True, exclude the last trade if the position was still open at the final bar (force-closed at bar_price).

  • raise_exceptions (bool) – If False, catch internal assertion errors and append them to Results.warnings instead.

  • use_numba (bool) – If True (default), use the Numba JIT engine. Set to False to use the reference Python engine.

Return type:

Results

Returns:

Results NamedTuple.

haymaker.research.auto_perf(data, price_column='open', slippage=1.5, skip_last_open=False, raise_exceptions=True, use_numba=True)[source]¶

Return performance statistics, converting raw signals if needed.

This is a convenience wrapper around perf() for notebook and exploratory workflows. It first checks whether data already satisfies the transaction-frame schema produced by no_stop() or stop_loss(). If so, it passes the dataframe directly to perf().

If data does not match that schema, the wrapper treats it as a raw strategy dataframe and tries to convert it through no_stop() using price_column. If neither path works, a ValueError explains both failures.

Parameters:

  • data (DataFrame) – Either a transaction-frame dataframe ready for perf(), or a raw dataframe accepted by no_stop().

  • price_column (str) – Price column used only for the fallback no_stop() conversion.

  • slippage (float) – Transaction cost expressed as a multiple of min tick.

  • skip_last_open (bool) – Passed through to perf().

  • raise_exceptions (bool) – Passed through to perf().

  • use_numba (bool) – Passed through to perf().

Return type:

Results

Returns:

Typical Pattern¶

from haymaker.research import no_stop, perf

strategy = df.copy()
strategy["signal"] = signal
strategy["position"] = strategy["signal"].shift().fillna(0).astype(int)

tx = no_stop(strategy, price_column="open")
result = perf(tx, slippage=1)
